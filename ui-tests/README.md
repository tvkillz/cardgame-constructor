Чтобы добавить новый сайт и сгенерировать для него снапшоты, вообще не нужно трогать старые тесты или существующие эталонные картинки. Playwright умеет сохранять и проверять снапшоты выборочно.

Для этого нужно выполнить два простых шага:
Шаг 1. Добавить сайт в массив sites

Просто допишите новый объект с именем и урлом в ваш массив sites в том же файле:
```typeScript

const sites = [
  { name: 'komorebi', url: 'http://127.0.0.1:3102' },
  { name: 'voidborn', url: 'http://127.0.0.1:3100' },
  { name: 'new-site', url: 'http://127.0.0.1:3103' }, // <--- Ваш новый сайт
];
```

Шаг 2. Запустить обновление снапшотов только для нового сайта

Чтобы Playwright не переписывал старые картинки для komorebi и voidborn, а сгенерировал эталоны только для нового сайта, запустите тесты с фильтром по имени (-g или --grep):
```bash

npx playwright test landing.spec.ts -g "new-site" --update-snapshots
```
